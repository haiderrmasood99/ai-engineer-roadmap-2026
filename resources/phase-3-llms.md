# Phase 3: LLMs

Outcome: build LLM systems that can be evaluated, monitored, and paid for.

## Learn

- prompting as interface design
- structured output and tool calling
- RAG ingestion, retrieval, chunking, reranking, and evaluation
- fine-tuning and when not to fine-tune
- agents, failure modes, and runaway loops
- cost architecture: caching, routing, batching, model selection

## Build

Build a production RAG system with hybrid retrieval, evals, and CI quality gates.

## Resources

- Simon Willison's writing on LLM systems
- Chip Huyen on ML systems and AI engineering
- OpenAI and Anthropic docs for current API patterns
- RAGAS documentation for evaluation ideas

## Skip

- RAG without evaluation
- agents before you understand basic tool reliability
- optimizing prompts while ignoring token cost
