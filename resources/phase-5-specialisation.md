# Phase 5: Specialisation

Outcome: choose a domain where your AI engineering is harder to replace.

## Paths

- Healthcare AI: safety, privacy, clinical workflows, explainability
- Fintech AI: model risk, audit trails, fraud, KYC, explainability
- AI system design: gateways, routing, caching, memory, cost controls
- MLOps: deployment, monitoring, data quality, platform reliability

## Build

Ship one serious domain project with README, tests, architecture diagram, evaluation, and failure modes.

## Resources

- Domain regulations and implementation guides
- Real job descriptions from target companies
- Public incident reports and postmortems
- Senior engineers writing from production experience

## Skip

- vague "AI expert" positioning
- domain projects with no domain constraints
